# 08_parser_impl

이 폴더의 목표는 **SQL 문자열을 실제로 해석하는 코드**를 만드는 것입니다.

앞에서 `parser.h`에서 파서가 무슨 일을 할지 약속했다면,
여기서는 그 약속을 실제 코드로 구현합니다.

## 이 단계에서 만드는 파일

- `parser.c`

## 이 파일이 하는 일

`parser.c`는 SQL 문자열을 받아서 `Command` 구조체를 채웁니다.

예:

```sql
INSERT INTO users VALUES ('Alice', '20');
```

이 문장이 들어오면:

- 명령 종류를 `COMMAND_INSERT`로 정하고
- 테이블 이름을 `users`로 읽고
- 값 `Alice`, `20`을 `values`에 저장하고
- `value_count`를 `2`로 채웁니다.

## 구현 포인트

- 공백을 건너뛰는 함수
- 키워드가 맞는지 검사하는 함수
- 테이블 이름을 읽는 함수
- `VALUES (...)` 안의 값을 나누는 함수
- 최종적으로 `parse_sql()`에서 INSERT/SELECT를 구분하는 흐름

## 이 파일을 볼 때 중요한 질문

1. 문자열에서 `INSERT`와 `SELECT`를 어떻게 구분하는가?
2. `users` 같은 테이블 이름은 어떻게 읽는가?
3. `Alice`, `20` 같은 값은 어디에 저장되는가?
4. 세미콜론(`;`)은 언제 검사하는가?
