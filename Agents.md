# Agent Working Rules

- When asking the user a question, explain the current step, why the question matters, and what a good answer should cover.
- Prefer one focused question at a time, but make it detailed enough that the user does not have to guess the context.
- For learning tasks, restate the file's role, the key design point being checked, and a concrete example when useful.
- When checking understanding, ask for the user's reasoning, not only the final conclusion.
- When implementing code for this learning project, add detailed inline comments so the user can follow what each important line is doing.
