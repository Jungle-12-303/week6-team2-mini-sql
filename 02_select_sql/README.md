# 02_select_sql

이 폴더의 목표는 `SELECT` 문장이 어떤 요청인지 이해하는 것입니다.

`INSERT`가 새로운 데이터를 넣는 문장이라면,
`SELECT`는 이미 저장된 데이터를 읽어서 보여 달라는 문장입니다.

## 이 단계에서 만드는 파일

- `select_materials.sql`

## 이 SQL이 의미하는 것

```sql
SELECT * FROM materials;
```

위 문장은 다음 뜻입니다.

- `SELECT`: 데이터를 조회하라.
- `*`: 모든 컬럼을 보여 달라.
- `FROM materials`: `materials` 테이블에서 가져와라.
- `;`: SQL 문장이 여기서 끝난다는 표시다.

## INSERT와 무엇이 다른가

- `INSERT`는 데이터를 저장한다.
- `SELECT`는 데이터를 읽어서 보여 준다.
- `INSERT`는 `VALUES`가 필요하다.
- `SELECT`는 어떤 테이블에서 읽을지만 알면 된다.

## 이 단계에서 꼭 설명할 수 있어야 하는 것

- 왜 `SELECT`는 넣는 문장이 아니라 읽는 문장인지
- 왜 `*`가 들어가는지
- 왜 `FROM materials`가 필요한지
- `SELECT`는 보통 어떤 결과를 화면에 보여 줄지
