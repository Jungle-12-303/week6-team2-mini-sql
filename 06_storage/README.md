# 06_storage

이 폴더의 목표는 **저장소 계층이 어떤 일을 맡는지 선언하는 것**입니다.

여기서 저장소(storage)는 실제 파일을 읽고 쓰는 부분입니다.
우리는 지금 DB 대신 `materials.csv` 같은 파일을 테이블처럼 사용할 것이기 때문에,
저장소는 결국 CSV 파일을 다루는 역할을 합니다.

## 이 단계에서 만드는 파일

- `storage.h`

## 저장소 계층이 하는 일

저장소는 SQL 문장을 직접 해석하지 않습니다.
그 일은 파서가 합니다.

저장소는 파서가 정리해 준 정보를 바탕으로 실제 파일을 다룹니다.

예를 들어:

- INSERT라면 -> CSV 파일 끝에 한 줄 추가
- SELECT라면 -> CSV 파일 내용을 읽어서 보여 주기

즉, 저장소는 "실제 저장과 실제 읽기"를 담당합니다.

## `storage.h` 안의 함수들

### `append_row_to_table`

역할:

- 어떤 테이블에 저장할지 받는다.
- 값 목록을 받는다.
- CSV 파일 끝에 새 줄을 추가한다.

예를 들어 `materials` 테이블에 `LX2 선바이저`, `ASD`, `MMH`, `21G`를 넣으라고 하면,
`materials.csv` 끝에 한 줄을 추가하는 느낌입니다.

### `print_table_rows`

역할:

- 어떤 테이블을 읽을지 받는다.
- 해당 CSV 파일을 읽는다.
- 파일 내용을 화면에 보여 준다.

즉, `SELECT * FROM materials;` 같은 요청이 들어오면,
`materials.csv`를 읽어서 출력하는 쪽입니다.

## 왜 함수가 두 개로 나뉘는가

읽기와 쓰기는 역할이 다르기 때문입니다.

- `append_row_to_table` -> 저장(쓰기)
- `print_table_rows` -> 조회(읽기)

이 둘을 나누면 코드가 덜 헷갈리고,
나중에 에러가 났을 때도 어디를 봐야 하는지 분명해집니다.
