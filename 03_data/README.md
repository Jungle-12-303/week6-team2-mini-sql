# 03_data

이 폴더의 목표는 **프로그램이 실제로 무엇을 저장하는지 눈으로 확인하는 것**입니다.

우리는 지금 진짜 DBMS를 만드는 것이 아니라,
아주 단순한 학습용 SQL 프로그램을 만들고 있습니다.
그래서 데이터베이스 대신 `CSV` 파일을 테이블처럼 사용할 것입니다.

## 이 단계에서 만드는 파일

- `materials.csv`

## CSV가 무엇인가

CSV는 값을 쉼표(`,`)로 나누어 저장하는 아주 단순한 텍스트 파일입니다.

예:

```text
LX2 sunvisor,ASD,MMH,21G
CN7 garnish,PPC,BLK,35G
```

여기서 한 줄은 보통 **한 개의 row(행)** 입니다.

## `materials.csv`를 어떻게 볼까

```text
LX2 sunvisor,ASD,MMH,21G
CN7 garnish,PPC,BLK,35G
```

이 파일을 이렇게 해석할 수 있습니다.

- 첫 번째 줄: `Alice`, `20`
- 두 번째 줄: `Bob`, `31`

즉, `materials` 테이블에 데이터가 두 줄 들어 있다고 볼 수 있습니다.

## 왜 지금은 DB 대신 CSV를 쓰는가

- 사람이 직접 열어 볼 수 있다.
- C 언어 파일 입출력 연습에 좋다.
- 저장 구조가 단순해서 흐름 이해에 집중할 수 있다.
- 나중에 `INSERT`는 한 줄 추가, `SELECT`는 전체 읽기라는 개념이 분명해진다.
