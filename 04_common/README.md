# 04_common

이 폴더의 목표는 **여러 파일이 함께 사용할 공통 약속을 만드는 것**입니다.

이제부터는 SQL 문장을 읽은 뒤,
그 결과를 어떤 모양으로 저장할지 정해야 합니다.
그 역할을 하는 파일이 `common.h` 입니다.

## 이 단계에서 만드는 파일

- `common.h`

## 왜 이 파일이 필요한가

나중에 프로그램은 크게 두 부분으로 나뉩니다.

- `parser.c`: SQL 문장을 읽고 분석한다.
- `executor.c`: 분석 결과를 보고 실제 동작을 실행한다.

그런데 두 파일이 서로 다른 방식으로 데이터를 이해하면 연결이 안 됩니다.
그래서 둘이 똑같이 보는 공통 구조체가 필요합니다.

## 이 단계의 핵심 개념

### 1. `CommandType`

이 값은 SQL 명령의 종류를 나타냅니다.

예:

- `COMMAND_UNKNOWN`
- `COMMAND_INSERT`
- `COMMAND_SELECT`

즉, 이 SQL이 INSERT인지 SELECT인지 구분하는 용도입니다.

### 2. `Command`

이 구조체는 파서가 SQL을 읽고 정리한 결과를 담는 상자입니다.

예를 들어 이런 정보가 들어갑니다.

- 명령 종류
- 테이블 이름
- INSERT라면 값 목록
- 값이 몇 개인지

## 왜 구조체가 필요한가

SQL은 처음에는 그냥 문자열입니다.

```text
INSERT INTO materials VALUES ('LX2 선바이저', 'ASD', 'MMH', '21G');
```

하지만 실행기는 이 긴 문장을 다시 해석하고 싶지 않습니다.
그래서 파서가 이 문자열을 분석해서 `Command` 구조체 안에 정리해 두면,
실행기는 그 구조체만 보고 쉽게 처리할 수 있습니다.
