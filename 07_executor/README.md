# 07_executor

이 폴더의 목표는 **실행기(executor)가 무엇을 하는지 선언하는 것**입니다.

실행기는 SQL 문장을 직접 읽지 않습니다.
그 일은 파서가 합니다.

실행기는 파서가 만들어 준 `Command` 구조체를 보고,
이 명령을 실제로 어떻게 처리할지 결정합니다.

## 이 단계에서 만드는 파일

- `executor.h`

## 실행기의 역할

실행기는 중간 연결 담당입니다.

예를 들어:

- `COMMAND_INSERT`라면 -> 저장 함수 호출
- `COMMAND_SELECT`라면 -> 조회 함수 호출

즉, 실행기는 "이 명령은 어디로 보내야 하지?"를 결정하는 역할입니다.

## 왜 실행기가 필요한가

파서와 저장소를 바로 붙여도 될 것처럼 보일 수 있습니다.
하지만 역할을 나누면 구조가 더 깔끔해집니다.

- 파서 -> 문장 해석 담당
- 실행기 -> 분기 담당
- 저장소 -> 실제 파일 처리 담당

이렇게 나누면 각 파일이 자기 책임만 가지게 됩니다.

## `execute_command`

이 함수는 `Command` 구조체를 받아서,
명령 종류에 따라 올바른 저장소 함수를 호출합니다.

예를 들어:

- `command.type == COMMAND_INSERT`
  -> `append_row_to_table(...)`
- `command.type == COMMAND_SELECT`
  -> `print_table_rows(...)`

만약 알 수 없는 명령이면 에러 메시지를 남깁니다.
